## 🐛 Common Issues and Fixes

### ❌ Executor won't launch / crashes immediately
**Fix:** Ensure your antivirus has whitelisted the Bunni folder. Windows Defender commonly blocks executor startup. Add an exclusion for the entire `BunniExecutor` folder, then relaunch as Administrator.

### ❌ Injection fails — "Failed to attach" error
**Fix:** Make sure Roblox is fully loaded into a game before clicking Inject (not just on the loading screen). If failure persists, check the **bunni executor discord** announcements channel — a Roblox update may have temporarily broken the bypass while a patch is prepared.

### ❌ Script Hub not loading / shows blank
**Fix:** This is a network issue. Disable VPN if active, check your internet connection, and restart Bunni. If the hub CDN is down, a status update will be posted in the Discord within minutes.

### ❌ Scripts execute but nothing happens in-game
**Fix:** Confirm the script is compatible with the current game version. Scripts from the built-in hub are regularly tested, but community-sourced scripts may be outdated. Try a hub script first to confirm your executor is working correctly.

### ❌ Windows Defender flags Bunni as malware
**Fix:** This is a false positive common to all executors — they use process injection techniques that AV tools flag by design. Add the Bunni folder to your Windows Defender exclusions. Community security reviews and **reddit bunni executor** discussions consistently confirm the tool is clean.

### ❌ Auto-Update not running
**Fix:** Ensure Bunni has internet access and isn't blocked by a firewall rule. Run the executor as Administrator — update checks require elevated permissions to write new files to the installation directory.

### ❌ "getgenv is nil" or globals error
**Fix:** The script you're running requires globals not supported by the current build. Check the UNC compatibility notes in the Script Hub entry for that script, or update Bunni to the latest version which improves UNC coverage.
